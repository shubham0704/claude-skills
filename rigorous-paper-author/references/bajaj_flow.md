# Bajaj / CVC-Inspired Flow Heuristic

This reference distills a recurring structure seen in theory-heavy and scientific-ML papers associated with the CVC orbit:

1. Start from a mathematically explicit formulation, not just a vague task statement.
2. Introduce the key reformulation or structured architecture early.
3. Insert a substantial preliminaries / notation section when the mathematics is nontrivial.
4. Separate the method from the theory rather than mixing all proofs into the algorithm section.
5. Make experiments answer theorem-adjacent questions whenever possible.
6. Use figures that carry explanatory load: architecture diagrams, diagnostic plots, side-by-side qualitative comparisons, or geometry/process diagrams.
7. End the introduction with contributions and a roadmap.
8. Put full proofs and long derivations in the appendix, but keep proof intuition in the main text.
